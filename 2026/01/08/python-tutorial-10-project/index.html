从零开始学Python（十）：实战项目 - 个人记账本

本系列教程将带你从零开始学习Python编程，无需任何编程基础。

学习目标

完成本章后，你将学会：

  • 如何将前面学到的知识综合运用
  • 如何设计一个完整的应用程序
  • 如何使用类组织代码
  • 如何使用文件存储数据

项目简介

我们将创建一个个人记账本应用程序，帮助你记录日常收入和支出，查看余额，并分析消费情况。

功能需求

  1. 添加交易：记录收入或支出
  2. 查看余额：显示当前账户余额
  3. 查看历史：显示所有交易记录
  4. 数据分析：按类别统计支出
  5. 数据持久化：保存到文件，下次启动时加载

技术要点

  • 类和对象：使用面向对象编程
  • 列表和字典：存储交易数据
  • 文件操作：保存和加载数据
  • 异常处理：处理用户输入错误
  • 控制流程：菜单循环和条件判断

项目设计

数据模型

每条交易记录包含：

  • 日期
  • 类型（收入/支出）
  • 金额
  • 类别
  • 备注

类设计

  1. Transaction：表示单条交易记录
  2. AccountBook：记账本主程序，管理所有交易

代码解析

Transaction 类

  • 属性：日期、类型、金额、类别、备注
  • 方法：
    • to_dict()：转换为字典，便于保存
    • from_dict()：从字典创建对象，便于加载
    • __str__()：定义字符串表示

AccountBook 类

  • 属性：交易列表
  • 方法：
    • add_transaction()：添加新交易
    • view_balance()：查看账户余额
    • view_history()：查看交易历史
    • analyze_expenses()：分析支出
    • save_data()：保存到JSON文件
    • load_data()：从JSON文件加载
    • run()：主循环

关键技术点

  1. 面向对象：使用类组织代码，提高可维护性
  2. 文件操作：使用JSON格式保存和加载数据
  3. 异常处理：处理用户输入错误和文件操作错误
  4. 列表操作：存储和遍历交易记录
  5. 字典操作：统计各类别支出
  6. 字符串格式化：美化输出显示

项目扩展

你可以继续扩展这个项目：

  1. 搜索功能：按日期、类别、关键词搜索交易
  2. 预算管理：设置月度预算，超支提醒
  3. 图表显示：使用matplotlib绘制收支图表
  4. 导入导出：支持CSV格式导入导出
  5. 多账户：支持多个账户（现金、银行卡等）
  6. 定期交易：自动添加每月固定收支

常见问题

Q1: 数据保存在哪里？

数据保存在 account_data.json 文件中，位于程序同一目录。

Q2: 如何备份数据？

直接复制 account_data.json 文件即可。

Q3: 数据丢失了怎么办？

如果没有备份，数据无法恢复。建议定期备份 JSON 文件。

Q4: 可以修改已添加的交易吗？

当前版本不支持修改。你可以添加一条新交易来冲销错误的记录。
